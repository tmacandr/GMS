Text tables contain strings of characters that are the names of geographic regions. In other words, text tables contain the names of continents, oceans, countries, etc. Text tables are considered regional data because the string of characters are confined to a well defined (i.e. bordered) area.

The generalized form of a text record is:
General Form of Text Record
  • ID =I,1,P,Row Identifier - Mandatory
  • *.TXT_ID =I,1,N,Text feature ID - Optional
  • [num-chars] =I,1,N,Number of chars in string - Mandatory
  • STRING =T,*,N,String of text - Mandatory
  • [num-coords] =I,1,N,Number of coords in array - Mandatory
  • SHAPE_LINE =C,*,N,Array of coordinates - Mandatory
Note that a text record is both a variant record as well as a variant length record. It is a variant record because of the optional Text-Feature-ID field. And it is a variant length record because the length of the String attribute is different for each record. Also, the Shape-Line is an array of coordinates whose length is different for each record.

A Minimum Bounding Rectangle (MBR) is a record that defines a region that contains a geographic entity such as a set of borders. An MBR record contains two coordinates (Latitude/Longitude) that define a rectangular region. A given geographic entity is then said to be contained within that rectangle.

A Node Table is essentially a point on the map. More formally, it is a zero-dimensional geometric primitive that is composed of a single coordinage tuple (pair or triplet) [Mil-Std-600006, pg 10].

There are two types of node primitives. The first is called an entity node. It is simply a point on the map. It is not dependent on any other geographic primitives. The second type is called a connected node. It is the point at the start or end of an edge (e.g. border). Therefore, a connected node cannot stand by itself. Also, since it is related to an edge, a connected node is paired with another connected node, thereby defining the start point and end point of an edge.

The general format of a node record is:
General Form of Node Record
  • ID =I,1,P,Row Identifier - Mandatory
  • *.PFT_ID =I,1,N,Id to Point Feature Tbl - Optional
  • CONTAINING_FACE =I,1,N,Fgn Key to Face Table - Only at Level 3
  • FIRST_EDGE =X,1,N,NULL - Optional
  • COORDINATE =C,1,N,Maximum Y Coord (Latitude) - Mandatory

Note that the above format is a variant record. The attribute Containging_Face is defined only at level 3 topology. Also, the *.PFT_ID attribute and the First_Edge attribute are optional.
