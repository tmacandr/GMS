9.1 - Objectives and Requirements

Under construction.

Table 4.1 : Requirements of GMS ADRG Graphics Toolkit
  1. Utilities to convert ADRG data to a buffer of pixel values.
  2. Assume the client software to be a raster graphics system (i.e. pixel based). No other assumption about the graphics package of the client shall be assumed. In other words, it shall not be assumed that the graphics toolkit of the client is GKS, nor MS Windows GDI, nor X Window System, etc.
  3. Utilities to generate a grid of lat/long lines shall be provided. These utilities shall:
    • allow the client to specify the number of latitude (i.e. parallel) and longitude (i.e. meridian) lines the grid is to contain
    • TBD
  4. Utilities to zoom in/out the screen coordinates shall be provided.
  5. The utilities shall be written in ANSI C++.
  6. The utilities shall be portable to the following platforms:
    • MS Windows 98
    • MS NT (ver xx)
    • HPUX UNIX (Hewlett-Packard)
    • Sun-Solaris UNIX
