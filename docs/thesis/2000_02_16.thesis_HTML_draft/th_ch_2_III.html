Chapter 2 - The DCW Database

The Types of Files in the DCW Database:

The files of the VPF database are classified into the following four categories:

  • Database Tables
  • Library Tables
  • Primitive Tables
  • Coverage Tables
The Database Tables contain general information about the whole VPF database (i.e. are not specific to a particular library). The Library Tables contain information about a specific regional library (NOAMER, SOAMAFR, EURNASIA, SASAUS) or the BROWSE library. The primitive tables contain low-level geographic information (such as borders of countries, outlines of continents, etc) for a given library (regional or BROWSE). The Coverage Tables contain descriptive information about a given region (such as soil conditions, maximum elevations, reefs, etc).

2.0 Basic Data-types of the VPF Database:

Each record consists of attributes of the following primitive data-types:

Table 2.x Primitive Datatypes
Data Type Description
T String of n characters
F 32-bit floating point
R 64-bit floating point
S 16-bit integer (i.e. short)
I 32-bit integer
C Two dimensional coordinate (x, y) (two 32-bit floats)
B Two dimensional coordinate (x,y) (two 64-bit floats)
Z Three dimensional coord (x, y, z) (three 32-bit floats)
Y Three dimensional coord (x, y, z) (three 64-bit floats)
D Date - 20 character string "YYYYMMDDhhmmss.xxxxx"
X NULL field (i.e. will be defined in future releases)
K Triplet-ID record (see section 7.0)
