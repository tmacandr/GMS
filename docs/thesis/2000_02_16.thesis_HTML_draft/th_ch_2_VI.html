Record Formats of Library Tables

2.5) Record Formats of Library Tables

This document describes the formats of the VPF files that describe the features of a given regional libary ... or the browse libary. These files are called Library Tables.

The files of the VPF database are divided into the following categories:

  • Database Tables
  • Library Tables
  • Primitive Tables
  • Feature Tables
  • Index Tables
  • Value-Description Tables
This document focuses on the library tables. The library Tables give description of a particular regional libary (or the browse library).

Each regional library (or the browse library) consists of the following tables:

2.0 Format of the Records From Library Tables:

This section describes the format of records from the library tables of the VPF database. The library files are:

2.3 LHT - Library Header Table

There are two library header tables on each CD. The first is from the regional libarary and the second is from the browse libary. The following record definition applies to both libaries (regional and browse).

  • ID=I,1,P,Row Identifier,-,-,:
  • PRODUCT_TYPE=T,12,N,Product Type,-,-,:
  • LIBRARY_NAME=T,12,N,Name,-,-,:
  • DESCRIPTION=T,100,N,Description of the library,-,-,:
  • DATA_STRUCT_CODE=T,1,N,Data Structure Code,-,-,:
  • SCALE=I,1,N,Denominator of the representative fraction,-,-,:
  • SOURCE_SERIES=T,15,N,Series,-,-,:
  • SOURCE_ID=T,30,N,Identifier of the source reference,-,-,:
  • SOURCE_EDITION=T,20,N,Edition number of the source,-,-,:
  • SOURCE_NAME=T,100,N,Name of library source,-,-,:
  • SOURCE_DATE=D,1,N,Source Date,-,-,:
  • SECURITY_CLASS=T,1,N,Security Classification,-,-,:
  • DOWNGRADING=T,3,N,Downgrading,-,-,:
  • DOWNGRADING_DATE=D,1,N,Date,-,-,:
  • RELEASABILITY=T,20,N,Releasability,-,-,:;

2.4 GRT - Geographic Reference Table

There are two Geographic Reference tables on each CD. The first is from the regional libarary and the second is from the browse libary. The following record definition applies to both libaries (regional and browse).
  • ID=I,1,P,Row Identifier,-,-,:
  • DATA_TYPE=T,3,N,Data Type,-,-,:
  • UNITS=T,3,N,Units,-,-,:
  • ELLIPSOID=T,15,N,Ellipsoid,-,-,:
  • ELLIPSOID_DETAIL=T,50,N,Ellipsoid Details,-,-,:
  • VERT_DATUM_REF=T,15,N,Datum Vertical Reference,-,-,:
  • VERT_DATUM_CODE=T,3,N,Vertical Datum Code,-,-,:
  • SOUND_DATUM=T,15,N,Sounding Datum,-,-,:
  • SOUND_DATUM_CODE=T,3,N,Vertical Datum Code,-,-,:
  • GEO_DATUM_NAME=T,15,N,Datum Geodetic Name,-,-,:
  • GEO_DATUM_CODE=T,3,N,Datum Geodetic Code,-,-,:
  • PROJECTION_NAME=T,20,N,Projection Name,-,-,:;

2.5 CAT - Coverage Attribute Table

There are two Coverage Attribute tables on each CD. The first is from the regional libarary and the second is from the browse libary. The following record definition applies to both libaries (regional and browse).

  • ID=I,1,N,Row Identifier,-,-,:
  • COVERAGE_NAME=T,8,P,Coverage name,-,-,:
  • DESCRIPTION=T,50,N,Coverage description,-,-,:
  • LEVEL=I,1,N,Topology level,-,-,:;

2.6 DQT - Data Quality Table:

There are two Data Quality tables on each CD. The first is from the regional libarary and the second is from the browse libary. The following record definition applies to both libaries (regional and browse).

  • EDITION_NUM=T, 8,N,Edition Number,-,-,:
  • CREATION_DATE=D, 1,N,Creation Date,-,-,:
  • REVISION_DATE=D, 1,N,Revision Date,-,-,:
  • SPEC_NAME=T, 20,N,Product Specification Name,-,-,:
  • SPEC_DATE=D, 1,N,Product Specification Date,-,-,:
  • EARLIEST_SOURCE=D, 1,N,Date of Earliest Source,-,-,:
  • LATEST_SOURCE=D, 1,N,Date of Latest Source,-,-,:
  • [num-chars]=I,1,num of chars in next variant length attribute:
  • QUANT_ATT_ACC=T, *,N,Standard Deviation of Quantitative Attributes,-,-,:
  • [num-chars]=I,1,num of chars in next variant length attribute:
  • QUAL_ATT_ACC=T, *,N,Percent Reliability of Qualitative Attributes,-,-,:
  • [num-chars]=I,1,num of chars in next variant length attribute:
  • COLLECTION_SPEC=T, *,N,Collection Specification Name,-,-,:
  • SOURCE_FILE_NAME=T, 12,N,Included Source File Name,-,-,:
  • [num-chars]=I,1,num of chars in next variant length attribute:
  • ABS_HORIZ_ACC=T, *,N,Absolute Horizontal Accuracy of VPF Level,-,-,:
  • ABS_HORIZ_UNITS=T, 20,N,Unit of Measure for Absolute Horizontal Accuracy,-,-,:
  • [num-chars]=I,1,num of chars in next variant length attribute:
  • ABS_VERT_ACC=T, *,N,Absolute Vertical Accuracy of VPF Level,-,-,:
  • ABS_VERT_UNITS=T, 20,N,Unit of Measure for Absolute Vertical Accuracy,-,-,:
  • [num-chars]=I,1,num of chars in next variant length attribute:
  • REL_HORIZ_ACC=T, *,N,Point to point horizontal accuracy of VPF Level,-,-,:
  • REL_HORIZ_UNITS=T, 20,N,Unit of Measure for Point to Point Hor. Acc.,-,-,:

  • [num-chars]=I,1,num of chars in next variant length attribute:
  • COMMENTS=T, *,N,Miscellaneous Comments,-,-,:;

2.7 LNT - Lineage Narrative Table:

There is only one Lineage.DOC file for each regional libary. A lineage.doc file is not part of the browse libary.

  • ID=I,1,P,Feature Table Primary Key,-,-,:
  • TEXT=T,80,N,Text Information,-,-,:;
